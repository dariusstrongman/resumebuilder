## Product framing (so the copy stays consistent)

**The diagnostic decides one of four paths and tells the user.** Not a preference picker.

- **STAY**: keep current job, no change
- **ADVANCE**: push for promotion or market-rate raise where you are
- **SPECIALIZE**: go deep on a high-premium niche inside current role
- **PIVOT**: jump to a higher-paying adjacent role

Verdict scoring is Mobley turnover model (1977, validated through 2026). Inputs are situational, not preferential. Highest-scoring path wins, with cited reasons.

Honesty rules still apply: no fabricated numbers, no roles the user can't realistically reach, every milestone has a source.

## Diagnostic intake (the 7 questions)

Replaced the 8-question preference form. Now situational only:

1. Current annual pay (USD numeric, drives BLS underpaid/overpaid check)
2. Last raise or promotion (lt12 / 12_24 / 24_48 / 48p)
3. Are you growing in this role (fast / some / plateau / backsliding)
4. Most weeks, how do you feel (energized / fine / drained / dreading)
5. Next title at current company (clear / unclear / capped / unknown)
6. Hours/week for skill-building (1 / 3 / 7 / 12)
7. Financial runway if you switched (gap_ok / cut_ok / continuous)

Resume is Step 1 (pasted or uploaded with PDF.js + mammoth), questions are Step 2. Each question has a "why we ask" hint.

## Verdict scoring (lives in the n8n Code node)

Function `scoreVerdict(intake, payCtx)` returns `{ stay, advance, specialize, pivot }` integers. Each input contributes weighted points to the 4 paths. Highest score wins. Reasons surfaced as 2-4 cited bullets (e.g., "you are underpaid by $Xk vs BLS median", "your role is capped here").

Tested live across 4 distinct intake patterns; each produces the expected verdict (STAY 12-5-3-(-8), ADVANCE 11-8-3-(-1), SPECIALIZE 10-8-4-1, PIVOT 18-8-1-(-10)).

## 3-horizon roadmap (built today)

Per verdict, generates 3 milestones with sourced pay + event:

- **STAY**: 6mo same, 1yr `current * (1+CAGR)`, 3yr `current * (1+CAGR)^3`
- **ADVANCE**: 6mo locks 70% of market gap if underpaid (else case-building phase), 1yr title bump compounds on top, 3yr two cycles
- **SPECIALIZE**: 6mo cert complete (no pay change), 1yr premium kicks in, 3yr senior with CAGR compounding
- **PIVOT**: 6mo cert + 30-50 apps (still in old job), 1yr in seat at expected pay, 3yr senior level (+30% on entry pay)

Cited research baked into the source strings:
- Ravio 2026 (promotion bumps + 4% annual rate)
- StudyTech 2026 (AWS cert: 60-68% first-attempt pass, 6-8 wks at 10-15 hrs/wk)
- boterview 2026 (tech job search 8-18 weeks, 100-200 apps to land an offer)
- Nectar 2024 (asking-for-promo conversation raises retention 41%)
- BLS OEWS May 2024 (per-role percentile bands)

## Stateful UX (the big "feels useful" change)

- `/career-outlook.html` loads `outlook_get` on entry. If active outlook exists, renders the existing report with task checkboxes, cadence selector, restart, cancel. Only shows the intake form if no active outlook.
- `/account.html` has a new "Career Outlook" section above Applications. Shows verdict ("Our pick: ADVANCE"), day-N-of-90 progress bar, current month's tasks with one-click did/wip/skip buttons that write through to `plan_status`.
- After fresh submit, frontend refetches `outlook_get` and renders the active view (with check-off + cadence + cancel) instead of the static report.

Persistence model: `career_outlooks` table on Supabase project `jjiizicwjldusuteffnb`. Columns: id, user_id, email, intake (jsonb), outlook (jsonb: pay_context + verdict + scores + roadmap + scenarios + current + top_adjacent), plan (jsonb), plan_status (jsonb keyed by task id), status_token, checkin_cadence, next_checkin_at, cancelled_at.
